# หลักการและเครื่องมือการติด Tag อัตโนมัติ
> เอกสารสำหรับทำสไลด์ — แต่ละหัวข้อ (`##`) ออกแบบให้เป็น 1 สไลด์

---

## สไลด์ 1 — ภาพรวม
ระบบติด tag เจตนา (intent) ให้แชต LINE OA **อัตโนมัติ** โดย
- ใช้ AI (LLM) วิเคราะห์บทสนทนา
- **อ้างอิงกฎจาก Rulebook เสมอ** (ไม่ปล่อยให้ AI เดาเอง)
- มี **คนยืนยัน** ก่อนใช้จริง (human-in-the-loop)

แนวคิดหลัก: *เร็ว + สม่ำเสมอ + ตรวจสอบได้ + ปลอดภัย*

---

## สไลด์ 2 — หลักการหลัก: RAG (Retrieval-Augmented Generation)
แทนที่จะถาม AI ตรงๆ (เสี่ยงมั่ว/hallucinate) เราทำ 2 ขั้น:

1. **Retrieval** — ดึง "กฎที่เกี่ยวข้อง" จากคลังเอกสาร (Rulebook) มาก่อน
2. **Generation** — ให้ AI ตอบ *โดยอิงเฉพาะกฎที่ดึงมา*

> ผลคือทุก tag มีที่มา (evidence) อ้างอิงได้ → ลดการมั่ว

---

## สไลด์ 3 — Pipeline การติด tag (7 ขั้นตอน)
```
[1] รับข้อความ LINE → Mask PII → เก็บ DB
        │
[2] รวมบทสนทนา (ที่ mask แล้ว) เป็นคำถาม
        │
[3] แปลงเป็นเวกเตอร์ (Embedding)
        │
[4] ค้นกฎที่ใกล้เคียง (Vector Search) + กรองสิทธิ์ + เวอร์ชันล่าสุด
        │
[5] สร้าง Prompt (กฎ + บทสนทนา) → ส่งให้ Gemini (โหมด JSON)
        │
[6] ตรวจรูปแบบผลลัพธ์ (Schema) + กฎความปลอดภัย (Fail-safe)
        │
[7] คนรีวิว → ยืนยัน/แก้/ปฏิเสธ → บันทึก Audit
```

---

## สไลด์ 4 — เทคนิคที่ 1: Embedding + Vector Similarity
- แปลงข้อความ → **เวกเตอร์ตัวเลข** (embedding) ด้วยโมเดล AI
- วัดความใกล้เคียงด้วย **Cosine Similarity** (ค่า 0–1)
- ดึง chunk ที่คะแนนสูงสุด (Top_K) มาเป็น context

> ทำให้ค้นกฎ "ตามความหมาย" ไม่ใช่แค่ keyword ตรงตัว

---

## สไลด์ 5 — เทคนิคที่ 2: Structured Output + Schema Validation
บังคับ AI ตอบเป็น **JSON ตายตัว** แล้วตรวจทุกครั้งก่อนบันทึก:

| ฟิลด์ | ความหมาย |
|---|---|
| `tags` | แท็กที่เลือก |
| `answer_summary` | สรุป ≤ 3 ประโยค |
| `evidence_list` | หลักฐาน (source/section/version) |
| `confidence_level` / `confidence_score` | ระดับ + คะแนนความมั่นใจ |
| `risk_flag` | สัญญาณเสี่ยง (เก่า/ขัดแย้ง/สิทธิ์) |

> ถ้ารูปแบบผิด/ขาด → ระบบไม่รับ (กัน hallucination)

---

## สไลด์ 6 — เทคนิคที่ 3: Fail-safe (กฎความปลอดภัย)
| สถานการณ์ | พฤติกรรมระบบ |
|---|---|
| มั่นใจต่ำ (score < 0.6) | **ปฏิเสธอัตโนมัติ** → ส่งเข้าคิวรีวิว |
| ข้อมูลขัดแย้งกัน | ติด `risk_flag = conflict` |
| ไม่พบกฎอ้างอิง | มั่นใจต่ำ + แจ้งใน assumptions |
| มีข้อมูลส่วนบุคคล | **Mask PII** ก่อนส่งเข้า AI |

---

## สไลด์ 7 — เทคนิคที่ 4: Human-in-the-loop
- AI **เสนอ** แท็ก — คน **ตัดสินใจ** (ยืนยัน/แก้/ปฏิเสธ)
- สุ่มตรวจ **5%** ของแท็กที่ผ่าน เพื่อ QA
- ทุกการกระทำถูกบันทึกใน **Audit Log** (ใคร/ทำอะไร/เมื่อไร)

> AI ช่วยเร่ง แต่คนยังเป็นผู้ควบคุมคุณภาพ

---

## สไลด์ 8 — เครื่องมือที่ใช้ (Tools / Tech Stack)
| ชั้น | เครื่องมือ | ใช้ทำอะไร |
|---|---|---|
| เว็บ + API | **Next.js 15** (App Router) + TypeScript | หน้าจอแอดมิน + REST API |
| UI | **Tailwind CSS** | ดีไซน์ |
| ฐานข้อมูล | **MongoDB** + Mongoose | เก็บแชต/แท็ก/กฎ/audit |
| AI ติดแท็ก | **Google Gemini** `gemini-2.5-flash` | วิเคราะห์ + ตอบ JSON |
| AI embedding | **Gemini** `gemini-embedding-001` | แปลงข้อความเป็นเวกเตอร์ |
| ค้นกฎ | Cosine similarity (in-app) / **Atlas Vector Search** | retrieval |
| ตรวจ schema | **Zod** | บังคับรูปแบบ JSON |
| ความปลอดภัย | **HMAC-SHA256** | ตรวจ LINE signature + session |
| ปกปิด PII | **Regex** | mask อีเมล/เบอร์/เลขบัตร |
| ช่องทางแชต | **LINE Messaging API** (webhook) | รับข้อความเข้าระบบ |

---

## สไลด์ 9 — เครื่องมือ map กับแต่ละขั้นตอน
```
รับข้อความ      → LINE Webhook API + Regex (mask PII)
เก็บข้อมูล      → MongoDB
embedding      → Gemini (gemini-embedding-001)
ค้นกฎ          → Cosine similarity / Atlas Vector Search
ติดแท็ก        → Gemini (gemini-2.5-flash, JSON mode)
ตรวจผลลัพธ์    → Zod schema
รีวิว/แสดงผล   → Next.js + Tailwind
```

---

## สไลด์ 10 — ทำไมออกแบบแบบนี้ (สรุปคุณค่า)
- **RAG** → ทุกแท็กมีหลักฐาน ลดมั่ว
- **Schema + Fail-safe** → ผลลัพธ์เชื่อถือได้ ไม่หลุดเกณฑ์
- **Human-in-the-loop + Audit** → ตรวจสอบย้อนหลังได้ คนคุมคุณภาพ
- **PII masking + Access control** → ปลอดภัย ตาม PDPA
- **โหมด mock** → สาธิต/พัฒนาได้โดยไม่ต้องมี API key

---

## ภาคผนวก — คำศัพท์
- **RAG**: ดึงข้อมูลจริงมาก่อนให้ AI ตอบ
- **Embedding**: ตัวเลขที่แทน "ความหมาย" ของข้อความ
- **Cosine similarity**: วิธีวัดความใกล้เคียงของเวกเตอร์ (1=เหมือน, 0=ต่าง)
- **Hallucination**: AI แต่งข้อมูลที่ไม่มีจริง
- **Human-in-the-loop**: มีคนอยู่ในวงจรตัดสินใจ
- **PII**: ข้อมูลระบุตัวบุคคล (เบอร์/อีเมล/เลขบัตร)
